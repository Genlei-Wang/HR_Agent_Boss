# 1. 背景

通过影刀搭建RPA应用，实现 BOSS 直聘平台候选人简历信息的全量实时抓取，并基于AP，对候选人的教育背景、工作经历、技能标签等核心信息进行结构化解析与深度语义分析。结合岗位需求构建的人才匹配度评估体系，系统可自动判定候选人与目标岗位的契合度，精准筛选出高匹配度人才，并智能触发定制化沟通话术，实现招聘流程的全链路自动化与智能化触达。

# 2. RPA设计
## 2.1 总体设计
基于飞书系统输出的岗位信息，构建智能化分组查询策略：通过动态分组算法，将大规模数据请求拆解为多批次、小颗粒度的查询任务。例如，当需批量获取 200 份候选人简历时，系统自动将任务拆分为 [80,1,50,34,2,17,5,10,1] 等子任务组，以分散数据调取压力。该机制有效规避平台风控检测阈值，通过流量错峰与请求分散策略，显著降低账号封禁风险，保障数据采集的稳定性与持续性。

#### 2.2.2.2 降低封控内容
在执行牛人信息采集任务时，采用智能循环抓取机制。每次切换至下一份信息采集前，系统将生成随机参数，依据该参数动态执行多维度风控规避策略：包括跳转至其他合规网页形成自然浏览轨迹、模拟真实用户行为进行页面随机滚动操作，以及在指定时间区间内随机设置请求间隔。通过这种动态化、拟人化的交互方式，有效混淆系统访问特征，规避平台反爬虫监测规则，降低因高频访问触发的账号封控风险，确保数据采集工作的持续稳定运行。
